# 프로젝트 결과 리포트 및 분석

## 1. MAC 연산의 이해와 AI에서의 중요성
MAC(Multiply-Accumulate) 연산은 입력 데이터(패턴)와 가중치(필터)의 대응되는 위치의 값들을 곱한 뒤, 그 결과들을 모두 더하여 하나의 스칼라 값을 도출하는 과정입니다. AI 분야, 특히 합성곱 신경망(CNN)에서 이미지의 특징을 추출할 때 핵심적으로 사용됩니다. MAC 연산을 통해 계산된 점수는 입력된 데이터가 특정 필터(패턴)와 얼마나 유사한지를 나타내는 '유사도' 역할을 하며, 점수가 높을수록 해당 특징이 강하게 존재함을 의미합니다.

## 2. 데이터 처리 및 정규화의 필요성
제공된 `data.json` 파일이나 실제 데이터 환경에서는 입력자의 의도는 같으나 표현 방식이 다른 데이터(예: `+`와 `cross`, `x`와 `X`)가 혼재할 수 있습니다. 프로그램 내부에서 이를 'Cross'와 'X'라는 표준 라벨로 정규화(Standardization)하면, 문자열 매칭 오류로 인한 논리적 실패를 방지하고 코드의 일관성과 유지보수성을 크게 높일 수 있습니다. 

## 3. 부동소수점 오차와 점수 비교 정책
컴퓨터는 실수를 이진법으로 표현하기 때문에, MAC 연산과 같이 수많은 실수를 더하고 곱하는 과정에서 미세한 부동소수점 오차가 누적됩니다. 만약 `score_a == score_b`처럼 직접 비교할 경우, 수학적으로는 동점이어도 오차 때문에 결과가 다르게 판정될 수 있습니다. 따라서 `abs(score_a - score_b) < 1e-9`와 같은 허용오차(epsilon) 정책을 적용하여 수치 비교 문제로 인한 판정 실패(FAIL)를 방지했습니다. 이 정책 덕분에 동점(UNDECIDED) 처리가 안정적으로 수행됩니다.

## 4. 성능 및 시간 복잡도 분석
각 패턴에 대해 MAC 연산을 수행할 때, 행렬의 크기가 N×N이라면 이중 반복문을 통해 정확히 N²번의 곱셈과 덧셈 연산이 발생합니다. 따라서 시간 복잡도는 O(N²)입니다.
성능 측정 결과표를 분석해 보면, 크기가 3x3(9번 연산)에서 25x25(625번 연산)로 커질 때 연산 횟수가 약 69배 증가하며, 측정된 소요 시간(ms) 또한 연산 횟수(N²)의 증가 폭과 거의 비례하여 가파르게 상승함을 확인할 수 있었습니다. 이는 O(N²) 알고리즘의 전형적인 특성입니다.

## 5. 실패 케이스(FAIL) 원인 진단 및 요약
이번 테스트에서 발생할 수 있는 실패 요인은 크게 세 가지로 진단할 수 있습니다.
- **데이터/스키마 문제:** JSON에서 명시된 N×N 크기와 실제 배열의 크기가 다른 경우(예: 필터는 5x5인데 패턴은 2x2인 경우). 프로그램이 다운되지 않도록 예외 처리 후 FAIL 및 사유를 로깅하도록 구현했습니다.
- **로직/라벨 문제:** expected 값이나 필터 키 값이 정규화되지 않아 매칭에 실패하는 경우. 표준화 로직을 통해 해결했습니다.
- **수치 비교 문제:** 상기 서술한 부동소수점 오차 문제이며 epsilon 적용으로 해결했습니다.
(결과적으로 정규화 및 epsilon 정책을 철저히 적용하여, 의도된 스키마 오류 케이스를 제외한 정상적인 연산 로직에서의 실패는 0건으로 통제되었습니다.)
